## 📖 О проекте

**Slinker** — это REST API сервис для сокращения длинных URL-адресов, разработанный в рамках учебного плана. Проект демонстрирует применение современных практик бэкенд-разработки на стеке **FastAPI + SQLAlchemy + PostgreSQL**.

### 🎯 Цели обучения

При разработке данного проекта решались следующие учебные задачи:
- ✅ Изучение принципов работы веб-фреймворка **FastAPI** (роутинг, зависимости, валидация).
- ✅ Практика работы с **асинхронностью** в Python.
- ✅ Освоение **паттерна Repository/Service** для разделения бизнес-логики и слоя данных.
- ✅ Настройка взаимодействия с базой данных через **SQLAlchemy ORM**.
- ✅ Реализация безопасного хранения конфигурации через **переменные окружения**.
- ✅ Практика написания чистой, типизированной документации и кода.

## 📦 Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/OtradinskiyIvan/slinker.git
cd slinker
```

### 2. Создание виртуального окружения

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# Linux/macOS
python3 -m venv venv
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

## ⚙️ Конфигурация

### 1. Настройка переменных окружения

Создайте файл `.env` в корне проекта на основе шаблона `.env.example`:

```bash
# Linux/Mac
cp .env.example .env

# Windows (CMD)
copy .env.example .env
```

Заполните файл `.env` актуальными данными для подключения к вашей локальной базе данных:

```env
URL_DB='postgresql://postgres:password@localhost:5432/slinker_db'
```

### 2. Инициализация базы данных

Создайте таблицы, выполнив скрипт инициализации:

```bash
python infrastructure/create_tables.py
```

### 3. Запуск сервера

```bash
# Через основной файл
python main.py

# Или напрямую через Uvicorn
uvicorn presentations.api_app:create_app --factory --reload --host 127.0.0.1 --port 8000
```

📚 **Документация API** будет доступна по адресу:
- Swagger UI: [http://localhost:8000/docs](http://localhost:8000/docs)
- ReDoc: [http://localhost:8000/redoc](http://localhost:8000/redoc)

## 📡 API Reference

### 1. Создать короткую ссылку

**POST** `/link`

```json
// Request
{
  "link": "https://github.com/OtradinskiyIvan/slinker"
}

// Response (200 OK)
{
  "link": "http://localhost:8000/x7k9m"
}
```

### 2. Редирект на оригинальную ссылку

**GET** `/{short_code}`

- `307 Temporary Redirect` — если ссылка найдена.
- `404 Not Found` — если короткий код не существует.

### 3. Просмотр статистики

**GET** `/{short_code}/statistics`

**Query Params:**
- `page` (int): Номер страницы (по умолчанию: 1)
- `size` (int): Записей на странице (по умолчанию: 20)

```json
// Response
{
  "page": 1,
  "size": 20,
  "total": 42,
  "items": [
    {
      "user_ip": "127.0.0.1",
      "user_agent": "Mozilla/5.0...",
      "used_at": "2026-04-07T15:00:00"
    }
  ]
}
```

## 🗄 База данных

### Схема данных

**Таблица `links`**
| Column | Type | Description |
|--------|------|-------------|
| `id` | Integer (PK) | Уникальный идентификатор |
| `real_link` | String | Исходный длинный URL |
| `short_link` | String (Unique) | Сгенерированный короткий код |
| `created_at` | DateTime | Дата создания записи |

**Таблица `usage_stats`**
| Column | Type | Description |
|--------|------|-------------|
| `id` | Integer (PK) | Уникальный идентификатор |
| `link_id` | Integer (FK) | Ссылка на `links.id` |
| `user_ip` | String | IP-адрес пользователя |
| `user_agent` | String | Строка User-Agent |
| `used_at` | DateTime | Время перехода |
